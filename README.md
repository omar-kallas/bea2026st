# BEA 2026 ST - Extended Implementation

This repository is taken from https://github.com/britishcouncil/bea2026st and based on it.

## Repository Structure

- `data/`: Original shared-task datasets split by `train/dev/test` and language (`es`, `de`, `cn`).
- `data_enriched/`: Feature-enriched versions of the datasets generated by `enrich_data.py`.
- `models/`: Saved model checkpoints and parameter CSV files:
  - `model_parameters.csv` for baseline models.
  - `num_model_parameters.csv` for numeric-feature models.
- `predictions/`: Model prediction CSVs organized by track, split, and language.
- `results/`: Evaluation summaries and detailed analysis outputs.
- `logs/`, `logs_sbatch/`: Training/prediction/evaluation logs.
- Core baseline scripts:
  - `download.py`
  - `finetune.py`
  - `predict.py`
  - `evaluate.py`
  - `run_pipeline.py`
  - `utils.py`

## Files Added In This Implementation

### `enrich_data.py`
Builds engineered features for each item and writes enriched CSVs under `data_enriched/`.

Implemented features include:
- IPA transcription for English target words and L1 source words.
- Frequency-based feature (`wordfreq` Zipf frequency).
- Morphological and length-based features (morpheme count, character length, syllables).
- Orthographic and phonological distance features (`Levenshtein` distance and phonetic edit distance).

### `finetune_num.py`
Adds a numeric-aware fine-tuning pipeline by extending transformer regression with extra numeric features.

Key additions:
- `NumericRegressionModel`: concatenates transformer CLS embedding with numeric features, then predicts with an FFN head.
- `NumericDataCollator`: batches text tensors + numeric feature tensors.
- Optional base-model freezing/unfreezing during early epochs.
- Separate learning-rate handling for encoder and FFN head.
- Numeric feature parsing, normalization metadata handling, and training integration through a custom trainer.

### `predict_num.py`
Runs inference for models trained with `finetune_num.py`.

Key behavior:
- Loads numeric-feature metadata stored in model config.
- Rebuilds numeric features from enriched data.
- Applies normalization using saved means/variances.
- Generates predictions and saves to the same prediction folder structure as baseline runs.

### `analyze_predictions.py`
Provides detailed post-hoc analysis of predictions against references.

Includes:
- Single-file and directory-wide analysis modes.
- Error analysis utilities (RMSE/correlation patterns, threshold-based summaries).
- Bucket tables and optional heatmap visualizations.
- Per-file detail reports and aggregate summary outputs.

## `_num` Options Added To `run_pipeline.py`

The pipeline now supports numeric-model training and inference with two added flags:

- `--finetune_num`: runs numeric-feature fine-tuning via `run_finetune_num(...)`.
- `--predict_num`: runs numeric-feature prediction via `run_predict_num(...)`.

Both options use the same general pipeline arguments (`--models_to_run`, `--model_params_path`, `--dataset_split`, `--seed`, `--verbose`).

## Example Usage

Run numeric fine-tuning then numeric prediction:

```bash
python run_pipeline.py \
  --finetune_num \
  --predict_num \
  --model_params_path models/num_model_parameters.csv \
  --dataset_split dev
```
